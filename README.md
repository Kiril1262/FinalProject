Telegram Bot Project

Опис:  
Це телеграм-бот для онлайн-магазину, який дозволяє користувачам:

- переглядати каталог товарів;
- оформлювати замовлення;
- залишати зворотний зв’язок;
- отримувати інформацію про допомогу, контакти та меню;
- адміністраторам — додавати нові товари через окрему панель.

Бот використовує Finite State Machine (FSM) для покрокової взаємодії з користувачем.

Структура проєкту:

- bot.py — головний файл запуску
- config.py — конфігураційні параметри
- init_db.py — ініціалізація бази даних
- .env — змінні середовища
- handlers/ — модулі для обробки команд
- database/ — робота з базою даних (наприклад, зворотній зв’язок)
- .bat — файл для запуску проєкту

Інструкція з запуску:

1. Клонуйте репозиторій на локальний комп’ютер.
2. Відкрийте папку з проєктом.
3. Запустіть проєкт подвійним кліком по файлу start.bat.

Увага:  
Ніяких додаткових налаштувань не потрібно — усе вже сконфігуровано (віртуальне середовище, залежності, змінні середовища, база даних тощо).

Programming Principles:

1. Single Responsibility Principle — кожен модуль має єдину відповідальність. Наприклад, catalog.py лише відображає каталог товарів.
2. Open-Closed Principle — систему можна розширити новими командами, не змінюючи існуючий код.
3. DRY (Don’t Repeat Yourself) — повторюваний код винесено у функції або окремі модулі.
4. KISS (Keep It Simple, Stupid) — логіка коду проста і зрозуміла.
5. Separation of Concerns — чітке розділення між обробкою команд, роботою з базою даних, логікою FSM та конфігурацією.

Design Patterns:

1. Command Pattern — реалізовано у папці handlers/, де кожна команда має окремий модуль (наприклад, help.py, feedback.py, catalog.py).
2. State Pattern — реалізовано у файлі order_fsm.py для обробки багатокрокового замовлення через FSM.
3. Factory Pattern — можна застосовувати для створення шаблонів повідомлень або товарів (у catalog.py чи в адміністративній частині), забезпечуючи однаковий стиль відображення.

Refactoring Techniques:

1. Метод вилучення (Extract Method) — довгі функції розбиті на менші, логічно завершені частини.
2. Вбудований метод (Inline Method) — усунення зайвих обгорток з непотрібною логікою.
3. Вилучення змінної (Extract Variable) — винесення проміжних обчислень у змінні з пояснюючими назвами.
4. Переміщення методу (Move Method) — методи перенесені у відповідні модулі для кращої організації коду.
5. Заміна магічних чисел константами — усі значення винесено у config.py або змінні середовища.
